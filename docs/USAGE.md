# DraftCopier 使用手冊 (USAGE)

**DraftCopier** 是一個可以幫助您將「Word (DOCX) 模板」與「Excel (XLSX) 清單」結合，並自動在 Gmail 中批次建立多筆草稿的自動化工具。

---

## 💡 核心運作原理

1. **準備模板：** 在 DOCX 檔案中，將需要替換的變數用雙大括號包起來，例如 `{{姓名}}`、`{{公司名稱}}`。
2. **準備資料：** 在 Excel 檔案中建立清單，第一列（標題列）必須包含對應的變數名稱（例如：`姓名`、`公司名稱`）。
3. **系統合併：** 系統會將 Excel 每一列的資料填入 DOCX 模板，同時保留 Word 中原有的粗體、底線、文字顏色等基本格式。
4. **生成草稿：** 透過 Google 帳號授權，系統會自動在您的 Gmail 「草稿匣」產生信件，等待您最後確認送出。

---

## 📝 準備檔案

### 1. Word 模板檔案 (.docx)
- 在信件內容中插入變數，格式為 `{{變數名稱}}`。
- 範例：
  ```
  親愛的 {{姓名}} 先生/小姐：
  
  感謝您參與本公司的專案。您的專屬報價單為 {{金額}} 元。
  ```
- **支援的格式**：支援文字的粗體 (Bold)、斜體 (Italic)、底線 (Underline)、刪除線 (Strikethrough) 以及文字顏色 (Color)。段落格式及圖片暫不支援。

### 2. Excel 資料清單 (.xlsx)
- 第一列必須是**標題列**，標題名稱必須與 Word 裡的變數完全一致（不包含大括號）。
- **【必備欄位】**
  為了讓系統知道要寄給誰以及主旨是什麼，您的 Excel 必須包含以下兩個欄位（名稱彈性很大，系統會自動模糊比對）：
  - **收件人信箱欄位**：可命名為 `email`、`e-mail`、`信箱`、`電子郵件` 等。
  - **信件主旨欄位**：可命名為 `subject`、`主旨`、`標題`、`信件主旨` 等。
- **【附件欄位】(僅限本機執行)**
  - 若需要夾帶附件，請在標題命名為 `附件`、`附件1`、`附件2`。
  - 欄位內容填寫「檔案名稱」（例如 `報價單.pdf`）。

## 🚀 操作流程

### 步驟 1：連結 Gmail 帳號
1. 進入網頁後，點擊「**連結 Gmail**」。
2. 按照畫面指示登入您的 Google 帳號。
3. 允許 DraftCopier 建立和管理草稿的權限。成功後畫面會顯示授權完成（或導回首頁）。

### 步驟 2：上傳檔案
1. 將準備好的 `.docx` 與 `.xlsx` 檔案拖曳到畫面中央的虛線框內，或是點擊按鈕手動選擇檔案。
2. 確認兩個檔案都已成功選取。

### 步驟 3：設定字型與附件（選用）
- **字型**：從下拉選單中選擇一種 Gmail 有支援的字型（如 Sans Serif、微軟正黑體等），確保收件人看到的字體不會跑版。
- **本機附件路徑**：如果您是使用**本機版**運行，可以在此欄位填入存放附件的實體資料夾路徑（例如：`/Users/you/Desktop/attachments`）。系統會根據這個路徑去找 Excel 裡填寫的檔名。*(註：雲端 Vercel 版本不支援本機附件)*

### 步驟 4：預覽
1. 點擊「**上傳並預覽**」按鈕。
2. 系統會立刻在下方顯示「第一筆資料」套用模板後的 HTML 結果，讓您確認格式、欄位名稱有沒有錯，文字顏色有沒有跑掉。
3. 如果出現「缺少必要欄位：email、subject」的紅色錯誤，請回頭檢查您的 Excel 標題列。

### 步驟 5：批次建檔並發送
1. 確認預覽畫面無誤後，點擊「**批次建立 Gmail 草稿**」。
2. 等待系統作業完成（會依照名單長度需要一些時間）。
3. 看到「已建立 Gmail 草稿：X 封」的成功訊息後，您可以 **開啟您的 Gmail 草稿匣**。
4. 草稿匣中現在應該躺著所有熱騰騰的信件，您可以打開確認它們、手動添加額外附件，最後點擊「傳送」發出。

## ⚠️ 常見問題與限制

1. **雲端版本不能上傳附件嗎？**
   是的。由於瀏覽器與伺服器的安全限制，Vercel 雲端版本無法讀取您電腦裡的實體檔案。如果您有嚴格的附件配對需求，建議在本地端執行此專案（`npm run dev`）。
2. **我可以放圖片進 DOCX 模板嗎？**
   目前轉換系統主要針對「文字及其樣式（顏色、粗體）」進行轉換。Word 裡面插入的圖片在轉換過程中會被丟棄。如果需要帶有重要圖片的信件，建議在生成草稿後去 Gmail 裡手動貼上圖片。
3. **為什麼會出現 "401 Missing session user_key"？**
   通常是因為您的登入狀態失效或逾時。請試著重新整理網頁，然後再點一次「連結 Gmail」按鈕重新授權即可。
